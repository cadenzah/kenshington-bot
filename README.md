# 켄싱턴 리조트 예약 모니터

켄싱턴 리조트 분양회원 예약 페이지의 객실 가용성을 주기적으로 조회하는 로컬 모니터링 도구입니다.

## 요구사항

- Node.js 20.6.0 이상
- 켄싱턴 리조트 분양회원 계정

## 설치

```bash
git clone <repo-url>
cd kenshington
npm install
```

## 초기 설정

```bash
cp .env.example .env
```

`.env`를 열고 값을 채웁니다.

```
MEMB_NO=015-XXXXX-XX      # 분양 회원번호 (필수)
MEMB_ID=your_login_id     # 켄싱턴 로그인 아이디 (권장)
MEMB_PW=your_password     # 켄싱턴 로그인 비밀번호 (권장)
PORT=3000                  # 서버 포트 (선택, 기본 3000)
```

> `MEMB_ID` / `MEMB_PW`를 설정해두면 세션이 만료될 때 서버가 Chrome을 자동으로 열어 재로그인합니다.  
> 설정하지 않으면 세션 만료 시 모니터링이 중단됩니다.

## 실행

```bash
npm start
```

브라우저에서 `http://localhost:3000` 접속

## 사용 방법

1. **지점** 드롭다운에서 원하는 리조트 선택
2. **체크인 / 체크아웃** 날짜 입력
3. **성인 / 아동** 인원 입력
4. **체크 간격** 설정 (기본 5분, 실제 대기는 ±20% 범위에서 무작위화)
5. **모니터링 시작** 클릭

입력한 폼 값은 자동으로 브라우저에 저장되어 다음 방문 시 유지됩니다.

### 모니터링 로그

| 색상 | 의미 |
|------|------|
| 초록 | 예약 가능한 객실 발견 (객실명·가격 표시) |
| 회색 | 빈 방 없음 |
| 빨강 | 오류 또는 세션 만료 |

## 세션 관리

모니터링 중에는 API 응답에서 세션 쿠키(`khrf`)가 자동 갱신됩니다.  
세션이 만료되면 아래 순서로 자동 복구를 시도합니다.

1. **Puppeteer 자동 재로그인** — `MEMB_ID` / `MEMB_PW`가 설정된 경우, Chrome을 열어 SSO 로그인을 자동화합니다.
2. **복구 실패 시** — 모니터링이 중단되고 로그에 세션 만료가 표시됩니다.

자동 재로그인이 시작되면 Chrome 창이 잠깐 열렸다 닫힙니다.

### UI에서 수동 갱신

페이지 하단 **세션 설정**을 펼쳐 아이디/비밀번호를 입력하고 **저장**하면 이후 세션 만료 시 자동 재로그인에 사용됩니다.

### 비밀번호 보관 정책

| 입력 경로 | 저장 위치 | 유지 기간 |
|-----------|-----------|-----------|
| `.env` 파일 | 디스크 (평문) | 파일을 직접 삭제할 때까지 |
| UI 세션 설정 | 서버 메모리만 | 서버 재시작 시 소멸 |

UI를 통해 입력한 비밀번호는 브라우저(localStorage) 또는 파일에 저장되지 않습니다. 서버 프로세스를 종료하면 사라집니다.
