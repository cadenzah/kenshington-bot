# 켄싱턴 리조트 예약 모니터

켄싱턴 리조트 분양회원 예약 페이지의 객실 가용성을 주기적으로 조회하고, 빈 방이 생기면 macOS 알림으로 알려주는 로컬 모니터링 도구입니다.

## 요구사항

- Node.js 20.6.0 이상
- 켄싱턴 리조트 분양회원 계정

## 설치

```bash
git clone <repo-url>
cd kenshington
npm install
```

## 초기 설정

### 1. 환경변수 파일 생성

```bash
cp .env.example .env
```

`.env`를 열고 아래 두 값을 채웁니다.

```
MEMB_NO=015-XXXXX-XX   # 분양 회원번호
KHRF=xxxxxxxxxxxxxxxx   # 세션 쿠키 (아래 참고)
PORT=3000               # 서버 포트 (선택)
```

### 2. KHRF 쿠키 가져오기

`khrf`는 로그인 세션을 식별하는 쿠키입니다.

1. Chrome에서 [켄싱턴 예약 페이지](https://www.kensington.co.kr/reservation/quick_member)에 로그인
2. **F12 → Application → Cookies → `www.kensington.co.kr`**
3. `khrf` 항목의 Value 복사 → `.env`의 `KHRF`에 붙여넣기

> 초기 설정 시에만 필요합니다. 모니터링 중에는 API 응답마다 쿠키가 자동 갱신되므로 별도 관리가 불필요합니다.

## 실행

```bash
npm start
```

브라우저에서 `http://localhost:3000` 접속

## 사용 방법

1. **지점** 드롭다운에서 원하는 리조트 선택
2. **체크인 / 체크아웃** 날짜 입력
3. **성인 / 아동** 인원 입력
4. **체크 간격** 설정 (기본 5분, 실제 대기는 ±20% 범위에서 초 단위로 무작위화)
5. **모니터링 시작** 클릭

입력한 폼 값은 자동으로 브라우저에 저장되어 다음 방문 시 유지됩니다.

### 모니터링 로그

| 색상 | 의미 |
|------|------|
| 초록 | 예약 가능한 객실 발견 (객실명·가격 표시) |
| 회색 | 빈 방 없음 |
| 빨강 | 오류 또는 세션 만료 |

객실이 발견되면 **macOS 데스크탑 알림**이 함께 전송됩니다.

## 세션 만료 시 대처

모니터링 중에는 쿠키가 자동 갱신되므로 세션이 유지됩니다. 로그에 **세션 만료** 메시지가 뜨는 경우는 서버를 2시간 이상 중단했다가 재시작할 때뿐입니다.

**방법 A — UI에서 갱신 (재시작 불필요)**
1. 페이지 하단 **세션 설정** 펼치기
2. 새로운 `khrf` 값 입력 후 **저장**

**방법 B — .env 수정 후 재시작**
1. `.env`의 `KHRF` 값 교체
2. `npm start` 재시작
